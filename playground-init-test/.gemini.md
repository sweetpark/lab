# Playground Init Test Context

This module is an automation tool for loading initialization data (defined in JSON) into a database. It is used for setting up development and test environments with dynamic variable replacement, data preprocessing (encryption, date handling), and table-specific processing strategies.

## Key Components

- **ToolRunner**: Main execution entry point. Orchestrates the flow from global context to CPID, MID, and payment data.
- **InitDataAssembler**: Assembles necessary data from properties and the database.
- **MetaDataProcessStrategy**: Strategy pattern implementation for handling different types of data (Merchant, Affiliate, Contract, etc.).
- **FileReadService**: Parses JSON templates and performs preprocessing (encryption, date formatting).
- **DBProcessService**: Executes the final SQL `INSERT ... ON DUPLICATE KEY UPDATE` statements.

## Project Conventions

- **Data Definition**: JSON files in `src/main/resources/data/`.
- **Variable Replacement**: Uses `${VARIABLE}` syntax replaced by `DataVariableResolver`.
- **Encryption**: Integrated with KMS using `kmsClient-1.0.0.jar`. Supported types: `ENC_HASH`, `OTP`, `PASSWORD`, `ENC_BASE64`.
- **Database**: MariaDB.

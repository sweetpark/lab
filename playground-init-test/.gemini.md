# Playground Init Test 컨텍스트

이 모듈은 JSON으로 정의된 초기화 데이터를 데이터베이스에 로드하는 자동화 도구입니다. 동적 변수 치환, 데이터 전처리(암호화, 날짜 처리), 테이블별 처리 전략을 통해 개발 및 테스트 환경 구축에 사용됩니다.

## 주요 컴포넌트

- **ToolRunner**: 실행 진입점. Global Context부터 CPID, MID, 결제 데이터까지의 흐름을 조율합니다.
- **InitDataAssembler**: Properties 및 DB에서 필요한 데이터를 조립합니다.
- **MetaDataProcessStrategy**: 데이터 타입(가맹점, 제휴사, 계약 등)에 따른 처리 전략 패턴 구현체입니다.
- **FileReadService**: JSON 템플릿을 파싱하고 전처리(암호화, 날짜 포맷팅)를 수행합니다.
- **DBProcessService**: 최종적으로 `INSERT ... ON DUPLICATE KEY UPDATE` 구문을 실행합니다.

## 프로젝트 컨벤션

- **데이터 정의**: `src/main/resources/data/` 경로의 JSON 파일.
- **변수 치환**: `${VARIABLE}` 구문을 사용하며 `DataVariableResolver`에 의해 치환됩니다.
- **암호화**: `kmsClient-1.0.0.jar`를 통한 KMS 연동. 지원 타입: `ENC_HASH`, `OTP`, `PASSWORD`, `ENC_BASE64`.
- **데이터베이스**: MariaDB 사용.

## 최근 변경 사항 및 이력

### 2026-03-31 (제안된 전략: 데이터 객체화)
- **문제 정의**: 하드코딩된 JSON 데이터와 복잡한 의존성 관리(GID -> VID -> MID)로 인한 유지보수 어려움 발생.
- **리팩토링 목표**: 데이터 구조(JSON)와 비즈니스 로직(Java 객체)을 분리하여 유연한 데이터 조립 및 자동 행(Row) 생성 구현.
- **제안된 아키텍처**:
    - **컨텍스트 기반 연결**: `GidContext`, `VidContext`, `MidContext`를 참조 기반 의존성으로 구현하여 ID 전파 자동화.
    - **Skeleton & Patch 패턴**: JSON은 '뼈대(컬럼 정의)'로만 사용하고, 자바 객체와 Enum을 사용하여 값을 동적으로 패치(Patch).
    - **Enum 기반 설정**: 매직 넘버(상태 코드, 서비스 타입 등)를 자바 Enum으로 대체하여 데이터의 의미를 명확히 함.
    - **행(Row) 생성 로직**: 단일 JSON 행 템플릿을 기반으로 자바 루프와 Enum을 사용하여 여러 행(예: `TBSI_MBS_SVC`)을 동적으로 생성.
    - **동적 케이스 지원**: `MerchantProfile`을 도입하여 JSON 파일 복사 없이 다양한 MID 데이터 변형(예: 정상/정지, 서비스 세트 다름)을 쉽게 생성.

### 2026-03-08 (유지보수 및 리팩토링)
- **버그 수정**: `TemplateContext` 생성 시 `division` 값 누락으로 인한 `RuntimeException` 해결.
- **디버깅 강화**: `TemplateContext`에 `toString()`을 추가하여 예외 발생 시 상세 정보 확인 가능하도록 개선.
- **결제 데이터 모듈화**:
    - TID 생성 및 `PayContext` 조립 로직을 `PayProcessService`로 분리.
    - 가맹점 정보를 캡슐화한 `PayBaseContext`를 도입하여 데이터 생성 표준화.
- **ToolRunner 리팩토링**:
    - `test.mode` 설정을 통한 실행 분기 처리 지원 (`FULL` vs `PAY`).
    - 결제 데이터 전용 모드를 위해 외부 설정(`pay.ptnCd`, `pay.mid` 등) 연동 기능 추가.
- **검증 로직 통합**:
    - `spring-boot-starter-validation` 의존성 추가 및 `@Validated`를 통한 필수 설정값 검증 강화.
